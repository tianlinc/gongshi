# RDM 工时填报自动化工具

一个用于自动填写 RDM 研发管理平台工时的 Python 工具。

## 功能特点

- ✅ 自动登录（AES 加密认证）
- ✅ 交互式工时填写
- ✅ 支持按周或按月填报
- ✅ 任务模板配置
- ✅ 工时预览和确认

## 安装依赖

```bash
pip install pycryptodome requests pyyaml
```

## 使用方法

### 方法 1: 交互式填报

直接运行程序，按提示输入：

```bash
python rdm_timesheet.py
```

### 方法 2: 使用配置文件

编辑 `timesheet_config.yaml`，配置你的任务模板，然后运行：

```bash
python rdm_timesheet.py --config timesheet_config.yaml
```

## 配置文件说明

`timesheet_config.yaml` 包含以下配置：

```yaml
# 登录信息
username: your_username
password: your_password

# 默认工作日（1=周一, 7=周日）
work_days: [1, 2, 3, 4, 5]

# 任务模板
tasks:
  - name: "项目开发"
    hours: [8, 8, 8, 8, 8, 0, 0]  # 每天工时
    completion_rate: 100
```

## 操作流程

1. **登录系统** - 使用用户名密码自动登录
2. **选择操作** - 选择填写本周、指定周或整月工时
3. **输入任务** - 逐个录入任务名称、每天工时、完成率
4. **预览确认** - 查看工时汇总后确认提交
5. **完成提交** - 自动提交到 RDM 系统

## 注意事项

⚠️ **当前版本为框架代码，需要根据实际系统调整以下内容：**

1. **API 端点** - 需要探测实际系统的工时提交 API
2. **数据格式** - 需要根据系统要求的格式提交数据
3. **任务获取** - 需要实现从系统获取当月任务列表的功能

## 调试模式

查看实际的 HTTP 请求：

```bash
python rdm_timesheet.py --debug
```

这会打印所有的请求和响应，帮助分析系统接口。

## 常见问题

**Q: 登录失败怎么办？**
A: 检查用户名密码是否正确，系统可能需要验证码或其他安全验证。

**Q: 如何找到实际的 API 端点？**
A: 在浏览器中打开开发者工具（F12），切换到 Network 标签，手动填写工时并提交，查看发送的请求。

**Q: 工时提交失败怎么办？**
A: 运行调试模式查看详细错误信息，可能需要调整数据格式或添加额外字段。

## 技术说明

- **加密方式**: AES-ECB 模式，密钥为固定字符串
- **会话管理**: 使用 requests.Session 保持登录状态
- **数据格式**: JSON 格式提交工时数据
