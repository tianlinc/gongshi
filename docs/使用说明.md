# RDM 工时填报系统 - 使用说明

## 系统需求

1. **Python 3.8+** - 确保已安装 Python
2. **网络访问** - 能够访问 `http://10.111.36.3:2029`
3. **账号密码** - 你的 RDM 系统登录凭据

## 快速开始

### 第一步：安装依赖

双击运行 `install.bat` 或手动执行：

```bash
pip install flask flask-cors requests pycryptodome beautifulsoup4
```

### 第二步：启动系统

双击运行 `start_v2.bat` 或执行：

```bash
python app_v2.py
```

### 第三步：访问系统

打开浏览器，访问：

```
http://localhost:5000
```

## 使用流程

### 1. 登录系统

- 在登录页面输入你的用户名和密码
- 点击"登录"按钮
- 系统会自动处理 AES 加密认证

### 2. 查看任务

登录成功后，系统会自动显示：
- 当前周信息
- 本周的日期列表

点击"同步任务"按钮，系统会从 RDM 系统拉取：
- 我负责的任务
- 我参与的任务

### 3. 填写工时

对于每个任务：
1. 输入任务名称（或从同步的任务中选择）
2. 填写每天的工作小时数（0-24小时，支持0.5小时）
3. 设置任务完成进度（0-100%）

### 4. 快速操作

- **一键填充** - 点击"工作日8小时"自动填充周一至周五各8小时
- **保存草稿** - 暂存当前填写的内容到浏览器
- **加载草稿** - 恢复上次保存的内容
- **清空重填** - 清除所有已填写的数据

### 5. 提交工时

1. 填写完成后，点击"预览"按钮
2. 查看工时汇总表格和总计
3. 确认无误后，点击"提交工时"
4. 系统会自动提交到 RDM 系统

## 高级功能

### 自定义任务模板

编辑 `templates_v2/dashboard.html` 中的 `loadTemplate()` 函数：

```javascript
function loadTemplate() {
    tasks = [
        { id: 1, name: "项目开发", hours: [8,8,8,8,8,0,0], rate: 100 },
        { id: 2, name: "代码评审", hours: [0,0,2,0,0,0,0], rate: 100 },
        // 添加更多任务...
    ];
    renderTasks();
}
```

### 修改服务器端口

编辑 `app_v2.py`，找到最后一行：

```python
app.run(debug=True, host='0.0.0.0', port=5000)
```

修改 `port=5000` 为其他端口，如 `port=8080`

### 禁用调试模式

将 `debug=True` 改为 `debug=False`，适用于生产环境。

## 常见问题

### Q: 登录失败怎么办？

**A**: 检查以下项目：
1. 确认用户名密码是否正确
2. 确认能否访问 `http://10.111.36.3:2029`
3. 查看控制台错误日志
4. 尝试在浏览器中手动登录，验证账号是否正常

### Q: 无法同步任务？

**A**: 当前版本使用示例数据。要获取真实任务，需要：
1. 分析 RDM 系统的任务查询接口
2. 更新 `app_v2.py` 中的 `get_my_tasks()` 方法
3. 实现实际的 HTTP 请求和数据解析

### Q: 工时提交失败？

**A**: 当前版本为演示模式。要实现真实提交：
1. 使用浏览器 F12 开发者工具
2. 手动提交一次工时
3. 查看 Network 标签中的请求详情
4. 更新 `app_v2.py` 中的 `submit_work_log()` 方法

### Q: 如何查看实际的 API 接口？

**A**: 使用以下步骤：
1. 打开浏览器，按 F12 打开开发者工具
2. 切换到 Network 标签
3. 在 RDM 系统中手动操作（查看任务、提交工时）
4. 查看请求的 URL、方法和数据格式
5. 更新代码中的相应部分

## 技术架构

- **后端**: Python Flask
- **前端**: Bootstrap 5 + Vanilla JavaScript
- **认证**: AES-ECB 加密
- **会话**: Flask Session
- **HTTP 客户端**: requests

## 安全说明

⚠️ **注意**：
- 用户凭据在会话中存储，关闭浏览器后会话失效
- AES 加密密钥硬编码在代码中（生产环境应使用环境变量）
- 草稿数据存储在浏览器 localStorage 中
- 不要在公共网络中使用此工具
